# Output Retention

The phone demo produces local run outputs that may include camera images and location data.

## Generated Output

```text
droidcam-prototype/runs/
```

This folder is ignored by git.

## Keep

Keep outputs when:

- they are needed for thesis evidence
- they show a useful failure case
- they are approved sample data
- they are needed to debug dashboard behavior

## Delete

Delete outputs when:

- they include private images
- they include sensitive locations
- they are accidental test runs
- disk space is low

## Future Policy

When a backend exists, add:

- retention period
- sample anonymization
- export format
- backup rule
- deletion process
