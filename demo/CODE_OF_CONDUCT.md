# Code Of Conduct

This project is academic and prototype-oriented. Keep collaboration direct, respectful, and evidence-based.

## Expected Behavior

- Explain technical decisions clearly.
- Keep feedback about code and experiments, not people.
- Document assumptions.
- Share reproducible commands.
- Ask before deleting or replacing large artifacts.

## Unhelpful Behavior

- Hiding failing experiments.
- Renaming important files without updating docs.
- Treating model metrics as final without context.
- Making large changes without a verification path.

## Conflict Resolution

When there is disagreement:

1. State the concrete goal.
2. Compare options by demo impact, correctness, complexity, and reproducibility.
3. Choose the smallest reversible step.
4. Document the decision in the relevant docs file.
