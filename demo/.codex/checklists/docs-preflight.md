# Documentation Preflight Checklist

Use this before finalizing documentation edits.

## Path Accuracy

- Every command uses the actual folder name `droidcam-prototype`.
- PowerShell examples use `py`.
- Model paths match the repository.
- Generated outputs are described under `runs/`.

## Reader Flow

Docs should make sense for:

- A new teammate
- A thesis reviewer
- A future dashboard developer
- A coding agent

## Required Links

When adding a new major workflow, link it from:

- `README.md`
- `docs/README.md`
- the nearest section index

## Avoid

- Secret values
- Personal tokens
- Unverified package versions
- Claims that DINOv3 runs locally unless it has been verified
