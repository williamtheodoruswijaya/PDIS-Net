# Project Docs Skill

## Use When

Use this skill when creating or updating documentation, tutorials, contribution guides, Codex setup, or MCP notes.

## Documentation Map

```text
README.md
GETTING_STARTED.md
CONTRIBUTING.md
AGENTS.md
docs/
.codex/
.skills/
```

## Style

- Be concrete.
- Use actual project paths.
- Prefer PowerShell examples.
- Mention verification commands.
- Separate current behavior from future plans.
- Do not claim DINOv3 is locally runnable unless verified.

## Checklist

- Link new docs from an index.
- Keep folder names accurate.
- Update dashboard contract docs if output fields change.
- Keep generated outputs out of docs except as examples.
